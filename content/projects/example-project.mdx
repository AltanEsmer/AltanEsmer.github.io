Building a realtime collaboration tool from scratch means confronting a handful of hard problems at once: presence awareness, conflict resolution, and delivering sub-100 ms latency to users spread across multiple regions. This project started as an internal tool for coordinating deployments across a distributed engineering team and grew into a full-featured dashboard used daily by over 200 people.

The frontend is a Next.js 14 App Router application rendered at the edge. Each panel in the dashboard subscribes to a shared WebSocket channel via a custom React hook that handles reconnection backoff, deduplication, and optimistic updates. State is managed with Zustand, keeping the bundle lean and avoiding the overhead of a heavier state library for what is fundamentally a stream-driven UI.

## Architecture highlights

The backend runs on a small cluster of Node.js services behind a load balancer, with Redis Pub/Sub handling fan-out to all connected clients. A separate presence service tracks who is viewing which resource and emits lightweight heartbeat events every 5 seconds. The database layer uses Postgres for durable records and Redis for ephemeral session state.

```ts
// Subscribing to a realtime channel
const channel = useRealtimeChannel('dashboard:deployments', {
  onMessage: (event) => dispatch({ type: 'PATCH', payload: event.data }),
  onReconnect: () => dispatch({ type: 'REFETCH' }),
});
```

Latency benchmarks showed p99 event delivery at 68 ms for users in the same region and 210 ms cross-region — well within the threshold that feels instantaneous for this use case. The presence system adds minimal overhead thanks to a dedicated Redis keyspace with 10-second TTLs, avoiding expensive database writes for transient state.
